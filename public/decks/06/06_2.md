# 1限目のおさらい

### 画像生成AIは「再構成」
- コピペではない／ランダム性がある／1回が「重い」

### 4つの要素
- 主題／スタイル／構図／仕上げ・条件

### 使ったツール
- **ChatGPT**：自然文で会話しながら寄せる
- **Firefly**：4枚並べて揺れを見る／UIで比率指定

---

<!-- _class: lead -->

# 画像生成での「反復」のしかた

---

# 反復のテクニックは第5回と同じ

| 操作 | 例（画像版） |
|---|---|
| **追加** | 「窓の外を雨にする」「夕方の光に」 |
| **削除** | 「武器を外す」「背景の文字を消す」 |
| **置き換え** | 「ファンタジー → 近未来に」「水彩 → 線画に」 |
| **比較** | 「同じプロンプトで4枚出して並べる」 |

骨格はチャット型と同じ。**1か所ずつ、観察しながら**

---

# ツール別の反復のしかた

### ChatGPT
- 会話の流れで **追加指示** ができる
  - 「背景だけ暗くして、他はそのまま」
- 「他はそのまま」と明示するのがコツ

### Firefly
- プロンプトを **書き直して再生成**
- 1回4枚出るので、その中で揺れを観察してから次を決める
- 「参照画像」機能で前の画像のスタイルを引き継げる（今回は使わない）

---

# 画像生成ならではのコツ①
## 言葉で変えにくい部分がある

- 顔の特定パーツの微調整、手の指の数、左右の細かい差 など
- 何度出してもズレるところは：
  - **プロンプトでがんばらず**、後の回で扱う **画像編集・ペイントオーバー** で直す
  - 「言葉の限界」を見極めるのも訓練

---

# 画像生成ならではのコツ②
## 「変えたつもり」がランダム差のことも

- 1枚しか出していないと、変化が **プロンプトのせい** か **ランダムのせい** か区別できない
- 対策：
  - **同じプロンプトで2〜3枚** 出して「ベースの揺れ」を見る
  - その上で **1か所だけ** 変えて、また2〜3枚
- 育てる単位：
  - 1か所変更 → 数枚出す → 傾向を読む → 次の1か所

---

# 画像生成は「思考実験」ではなく「実験室」

仮説 → 試行 → 観察 → 修正 のサイクル

- 頭の中だけで完璧なプロンプトを作ろうとしない
- **出してから考える** が早い
- ただし投げる前に **何を確かめたいか** は決めておく

---

# 観察メモの例

| 観察する項目 | メモ例 |
|---|---|
| 思い通りだった点 | 銀髪、赤い瞳、バストアップは反映された |
| ズレた点 | マントが黒ではなく紫になった。背景が派手すぎた |
| 次に変える1か所 | 「背景は白でシンプル」を追加する |
| 変えない点 | 主題・構図・スタイルはそのまま |

→ このメモが **次のプロンプトの設計図** になる

---

# 比率・枚数・解像度の基本

---

# アスペクト比

- **縦長**（例：2:3 / 9:16）
  - 人物・ポートレート向き
- **横長**（例：3:2 / 16:9）
  - 背景・風景向き
- **正方形**（1:1）
  - SNS・アイコン向き

→ 出したい用途で決める。**主題と構図とセットで考える**

---

# ツール別：比率の指定方法

### ChatGPT
- **プロンプトに書く**：「正方形で」「縦長（9:16）で」
- 完全には効かないことがあるので、出てから確認

### Firefly
- 右パネルから **UIで選択** できる（1:1 / 4:3 / 3:4 / 16:9 / 9:16）
- 指定が確実に効く

---

# 枚数：複数枚が基本

- 1枚だけだと「揺れ」と「方向性」が見分けられない
- ChatGPT：1回に1枚 → 何度か投げて比べる
- Firefly：1回に **4枚** 出る → 1回でベース揺れを観察できる
- 良いのが出たら、そのプロンプトを **保存**
- ハズレも残しておくと「効かなかった指示」の記録になる

---

# 解像度：いきなり最高画質にしない

- 序盤は **中解像度で量** をこなす
  - 同じ時間で試行回数が増える＝感覚が早く育つ
- 方向が固まってから **良いものだけ高解像度** で出し直す
- 「最初から完成画」を狙うと、結局やり直しが多くなる

---

# 注意：細かい構図コントロールは次回

- 「もっとこの角度から」「カメラを引いて」などの精密コントロールは **第7回**
- 今日は **「狙った方向に出す」** ところまで

# ワーク②：自分のキャラを出してみる

---

# ワークの目的

- 第5回で作った **設定プロンプト** を、**画像生成プロンプト** に翻訳する
- 自分のキャラの絵を **初めて** 出す
- **1枚で決めない** ／ 反復で寄せる体験をする

---

# 翻訳の考え方

第5回のプロンプト（文章ベタ書き）から、4要素にあたる部分を抜き出して **`#` 構造** に書き直す

### 元の設定文（第5回）

```text
このキャラクターは、人見知りだが芯が強い魔法使いの少女です。
古い森に住んでいて、普段は静かですが、困っている人を見ると
放っておけません。少しミステリアスで、優しい雰囲気があります。
```

そのままだと **画像に出にくい言葉** がたくさん含まれている

---

# 画像に出にくい抽象語の例

- 人見知り
- 芯が強い
- 困っている人を見ると放っておけない
- ミステリアス
- 優しい雰囲気

→ これらは **見た目（色・形・表情・構図・光・服装）** に変換する必要がある

---

# 抽象設定 → 見た目に変換

| 抽象的な設定 | 見た目に変換した例 |
|---|---|
| 人見知り | 少し目線を外している、控えめな表情 |
| 芯が強い | 背筋を伸ばして立っている、しっかり杖を持つ |
| 古い森に住んでいる | 背景に深い森、苔のある木、古い本 |
| ミステリアス | 暗めの青緑の色調、柔らかい逆光 |
| 優しい雰囲気 | 穏やかな表情、丸みのあるデザイン |

---

# 翻訳後の画像生成プロンプト

```text
# 主題
10代の魔法使いの少女。深い森の中に立っている。
紺色のローブ、丸い帽子、古い本を抱えている。
背筋を伸ばして短い杖を持っている。

# スタイル
アニメ調、丸みのあるデザイン

# 構図
正面からのバストアップ、少し目線を外した控えめな表情

# 仕上げ・条件
暗めの青緑の色調、柔らかい逆光、
背景は森だが、人物が目立つようにシンプルにする
```

---

# Fireflyに入れるとき：自然文に直す

Fireflyは入力欄が1つ。`#` 見出しを外して **自然文** にすると安定

```text
深い森の中に立つ10代の魔法使いの少女。紺色のローブと丸い帽子、
古い本を抱え、短い杖を持っている。背筋を伸ばして、少し目線を外した
控えめな表情。正面からのバストアップ。アニメ調で丸みのあるデザイン。
暗めの青緑の色調、柔らかい逆光。背景は森だが、人物が目立つように
シンプルに。
```

ChatGPTには `#` 構造のまま貼ってOK
