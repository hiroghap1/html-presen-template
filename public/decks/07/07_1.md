# 今回のメインツール：Adobe Firefly（web版）

ブラウザで **firefly.adobe.com** にアクセス（Adobeの有料アカウントでログイン）

### 基本画面
- プロンプト入力欄（描きたいものを書く）
- 生成ボタン
- **設定パネル**：モデル／縦横比／解像度／参照画像／（Image 4のとき）効果・色とトーン・照明・カメラアングル など

---

# Fireflyの特徴①
## 権利面に配慮したAdobeモデル

- Adobe Stockなど、使用許諾のある素材を中心に学習
- 商用利用時に **説明しやすい** メリット
- ただし、文字・ロゴ風の装飾が出ないわけではない
  → 生成物は **必ず人間が確認** すること

---

# Fireflyの特徴②
## 構図・スタイルをGUIで選べる

- 縦横比・カメラアングル・参照画像を **画面の設定** で指定できる
- 効果（Effects）は仕上げ用
- **プロンプト×GUIの両刀** であやつる！

---

# Fireflyの特徴③
## ネガティブプロンプト専用欄は「ない」

- 他の画像生成AIにあるようなネガティブプロンプト専用欄がない
- → **プロンプト本文** で除外を表現する

# Fireflyでの「除外」の伝え方

---

# Fireflyは「プロンプト本文」で除外を書く

| ツールタイプ | 除外の指定方法 |
|---|---|
| Stable Diffusion系 | ネガティブプロンプト欄に出したくない要素を並べる |
| **Firefly（今回）** | **プロンプト本文に自然文で書く** |

### Fireflyでの書き方の例
- 「〜は写さない」
- 「人物は1人だけ」
- 「背景に文字を入れない」

---

# Markdownで考えて、自然文に流し込む

第6回で導入した `#` 見出しテンプレートは **考えを整理する下書き** として使う

Fireflyにはこの構造のまま入れず、**最後に自然文に流し込む**

→ 除外用に `# 描かないもの` を追加して整理する

---

# 下書き（Markdown構造）

```text
# 主題
10代の魔法使いの少女。紺色のローブ、丸い帽子、短い杖。

# スタイル
アニメ調、太めの輪郭線

# 構図
正面からのバストアップ

# 仕上げ・条件
色数を抑える、背景はシンプル

# 描かないもの
他の登場人物、ごちゃついた背景、余計な小物
```

---

# Fireflyに入れる自然文（フラット版）

```text
10代の魔法使いの少女。紺色のローブ、丸い帽子、短い杖。
アニメ調で太めの輪郭線。正面からのバストアップ。
色数を抑え、背景は無地でシンプル。人物は1人だけ。
```

> 構図（バストアップ・正面）は文章だけでなく **2限目でGUIからも指定** する。
> まずは「除外を言葉で言えること」を目標にする。

---

# 書き方のコツ ①②

### ① 具体的な名詞で
×「変なもの」 → ⭕️「他の人物」「余計な小物」「ごちゃついた背景」

### ② 盛り込みすぎない
一度に10個も足すと、何が効いたか分からない
→ **除外も1か所ずつ** が基本

---

# 書き方のコツ ③④

### ③ 引きすぎ注意
- 「背景なし」→ 人物が浮く
- 「影を消す」→ 立体感が消える
- 必要な要素まで巻き込むことがある

### ④ ポジティブと矛盾させない
本文で「2人」と書きながら「人物は1人」などの矛盾する指示をしない

---

# 書き方のコツ ⑤：否定が効かないときは肯定で言い換え

Fireflyは「○○を描かない」という **否定文が効きにくい場面** がある
→ そのときは **肯定文に言い換える** ことも試す

| 否定（効きにくいことがある） | 肯定への言い換え |
|---|---|
| 背景に文字を描かない | 無地のシンプルな背景 |
| ごちゃついた背景にしない | 余白の広い背景 |
| 影を強くしすぎない | やわらかい光、柔らかい影 |

# ワーク①
## 自分のキャラから「ノイズ」を引く

---

# ワークの目的

- 第6回で出した **自分のキャラ画像とプロンプト** を素材に使う
- 一から作らず、**既存のプロンプトを育てる**
- 「描かないもの」を言葉で扱える状態にする

### 使うモデル
**Firefly Image 4** に固定

---

# やること

### 1. 邪魔な要素を3つ書き出す（5分）
- 自分の画像を見て、「狙っていない要素」を **3つ言葉で**

### 2. 末尾に「〜にする／〜は描かない」でまとめる（5分）
- **肯定が基本**：「人物は1人だけ」「無地のシンプルな背景」
- 否定も可：「他の登場人物は描かない」（効かなければ肯定に言い換え）

### 3. **まず1つだけ** 除外を入れて再生成（10分）
- 変化を観察する／1か所ずつが原則

---

# やること（続き）

### 4. 順に1〜2個を足す／引きすぎたら戻す（10分）
- 効果を見ながら、少しずつ
- 「肯定への言い換え」も試す

### 5. 近くの人と共有する（5分）
- 入れた除外の言葉／効いた・効かなかった例／引きすぎ・言い換えが効いた例

---

# 観察メモの例

| 観察する項目 | メモ例 |
|---|---|
| 入れた除外の言葉 | 「人物は1人だけ」 |
| 効いた点 | 後ろにいた別人が消えた |
| 効かなかった点 | 手の指の本数は変わらなかった |
| 引きすぎた点 | 「背景なし」→ 人物が浮いた |
| 肯定への言い換え | 「無地の背景」に直したら整った |
| 次に試す1か所 | 「余計な小物は描かない」を追加 |

---

# 1限目のポイント

- Fireflyの除外は **プロンプト本文での引き算**
- 否定が効かないときは **肯定で言い換える**
- **1か所ずつ・観察しながら**

---

# 今日「あえて」やらないこと

- **生成消去**（要素をブラシで消す）
- **生成塗りつぶし**（部分的に作り直す）
- **生成拡張**（キャンバスを広げる）

### でも今日はプロンプトだけで戦う
- まず **言葉で何を引くか考える力** を鍛える
- 他のAIではプロンプトが重要！

→ 後修正は今後の授業で扱います
→ 2限目の最後に **少しだけ** のぞき見
