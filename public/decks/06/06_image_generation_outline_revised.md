# 第6回：はじめての画像生成

## 授業の概要

第5回までは「テキストでAIに伝える」訓練を積んできた。今回からは、いよいよ **画像生成AI** にプロンプトを渡し、テキストから絵を出してみる。

ここでは、第5回の5要素（役割・タスク・前提・制約・出力形式）と反復の考え方を、画像生成にも応用することを軸に据える。チャット型AIとの違い（役割指定が効きにくい・出力形式が「画像」に近い・1回の試行コストが高い等）を押さえつつ、画像プロンプト特有の構造として **主題／スタイル／構図／仕上げ・条件** を導入する。

最後に、第5回で作った設定プロンプトをベースに、画像生成用のプロンプトへ書き直し、**自分のキャラを1枚出力する** ところまで体験させる。ネガティブプロンプトや構図の精密コントロールは第7回に回し、本回は「狙った方向に出す」ことに集中する。

## 授業目標

1. 画像生成AIの仕組みを「テキストから画像が出る」レベルで直感的に説明できる
2. 画像生成プロンプトの基本要素（主題・スタイル・構図・仕上げ・条件）を意識して書ける
3. 第5回で作った設定プロンプトをベースに、自分のキャラの絵を初歩的に出力できる
4. 出力結果を観察し、1か所ずつ修正しながらプロンプトを改善できる

    - B. 主題・スタイル・構図・色・品質 → これは画像プロンプト寄り
    - C. 起・承・転・結・まとめ → 物語構成であってプロンプトの型ではない

  - Q3. プロンプトの修正でやりがちな失敗は？
    - A. 役割を指定する → 推奨される
    - B. 「いい感じに」「カッコよく」など抽象語の塊にする → **正解。** 解像度不足
    - C. 出力形式を指定する → 推奨される

#### 2. 画像生成AIの正体（約15分）

- 第1回でも触れた仕組みを、画像生成にフォーカスして復習
- ざっくりした流れ：
  - 大量の「画像 ＋ それを説明するテキスト」のセットを学習している
    - 例：「赤いワンピースの女の子」「夕焼けの海」「未来的な都市」といった言葉が、画像の中のどんな色・形・構図・雰囲気と結びつきやすいかを学んでいる
  - プロンプト（テキスト）を入れると、その言葉と関係の深い特徴をもとに、新しい画像を生成する
  - そのため、学習データに少ない概念・新しい言葉・曖昧な言葉は、うまく反映されにくいことがある

- 「過去の画像を検索して貼り合わせている」のではなく、学習した言葉と画像の関係をもとに生成している点を押さえる
- 第3回で扱った倫理の話と接続し、「学習」「模倣」「権利」「作家名指定」の問題を再確認する
- 重要なポイント：
  - **同じプロンプトでも、出るたびに違う絵が出る**（ランダム性）
  - **1回1回が完全なガチャではなく、傾向のコントロール**
  - **画像生成はチャットよりも時間・計算量のコストが高い**ため、1回投げる前に意図を整理する

> **Stable Diffusion**  
> テキストプロンプトをAIが扱える「意味の情報」に変換し、その情報を手がかりに、ノイズのような状態から少しずつ画像を生成するAIです。  
> 過去の画像を検索して貼り合わせているのではなく、大量の画像とテキストの関係から学習した特徴をもとに、プロンプトに合う新しい画像を作ります。  
> https://github.com/compvis/stable-diffusion  
> https://github.com/huggingface/blog/blob/main/stable_diffusion.md  
> https://openreview.net/pdf?id=M3Y74vmsMcY

#### 3. チャット型と画像生成、何が違う？（約15分）

| 観点 | チャット型 | 画像生成 |
|---|---|---|
| 入力 | テキスト | テキスト（＋画像） |
| 出力 | テキスト | 画像 |
| 役割指定 | 効きやすい（「専門家として」） | 効きにくい場合がある |
| タスク | 自由に指定できる | 多くの場合「画像を作る」に近い |
| 出力形式 | 箇条書き・表・HTMLなど自由に指定できる | 基本は画像。比率や枚数は別パラメータで指定することが多い |
| 反復のしやすさ | 言葉で細かく直せる | 言葉では微調整しづらい場面が多い |
| 1回のコスト | 比較的低い | 比較的高い（時間・計算量） |

- だから画像生成では：
  - プロンプトを **作り込んでから投げる**
  - **何が効いて何が効かないか** の感覚を早めに掴む
  - 出力結果を見て、言葉に戻して観察する

#### 4. 画像生成プロンプトの基本要素（約20分）

第5回の5要素は画像生成でも使える。ただし、画像生成では特に **主題・スタイル・構図・仕上げ・条件** の4要素が重要になる。

- **タスク** は「画像を生成する」に近くなる
- **前提** は「何のために使う画像か」「誰に見せる画像か」として残る
- **制約** は「避けたい表現」「背景」「文字の有無」「比率」「雰囲気」などとして残る
- **出力形式** は画像そのものだが、縦長・横長・正方形などの条件を指定できる場合がある

##### 画像生成プロンプトの4要素

1. **主題**（Subject） — 何を描くか
   - 例：「ファンタジー世界の10代の少女」
   - 例：「夕方の街角に立っている黒猫」
   - 例：「木製の机の上に置かれたりんご」

2. **スタイル**（Style） — どんな画風で
   - 例：「アニメ調」「水彩風」「フラットイラスト」「絵本風」「レトロなポスター風」
   - ※実在作家名・作品名は避ける（第3回ルール）

3. **構図・視点**（Composition） — どう写すか
   - 例：「バストアップ」「全身」「正面」「横顔」「俯瞰」「ローアングル」
   - 例：「中央配置」「余白を広く取る」「背景を大きく見せる」

4. **仕上げ・条件**（Quality / Details） — どんな仕上がりにするか
   - 例：「線画ベース」「色数を抑える」「柔らかい光」「背景はシンプル」
   - 例：「小学生向け教材に使えるように、怖すぎない表情」
   - 注意：「高品質」「最高傑作」「いい感じ」だけでは、何を改善したいのかが曖昧になりやすい

##### 第5回テンプレートとの違い：Markdown構造で書く

第5回ではチャット型AI用に `【役割】` のような **【】見出し** でテンプレートを書いた。第6回からは、プロンプト全体を **Markdownの `#` 見出し** で構造化する。

```text
# 主題
ファンタジー世界の10代の少女

# スタイル
アニメ調
```

**Markdownを使う理由：**

- LLM（ChatGPT、Claude など）は **Markdownを大量に学習** しており、`#` をセクション境界として正確に認識する
- `【】` は日本語圏ローカルな慣習で、AIによっては構造の読み取りが安定しないことがある
- 画像生成AIに直接渡しても、`#` は害になりにくい
- 第5回までの「テキスト用テンプレ」から、画像用に **より構造化された記法** に切り替える節目として導入する

> ツールによっては、最終的に `#` 見出しを外して **自然文 or カンマ区切り** に整理してから入力することもある（ツール側の慣習に合わせて使い分け）。

##### 実在作家名・作品名を避けるための言い換え例

| 避けたい指定 | 代わりに使える表現 |
|---|---|
| ○○先生風 | 太い輪郭線、淡い色、柔らかい陰影、繊細な線画 |
| ○○の作品風 | セル画風、絵本風、レトロなポスター風、フラットイラスト |
| 有名アニメ映画風 | 手描き感のある背景、温かい色調、やわらかい光、冒険感のある構図 |
| 有名ゲーム風 | ドット絵、厚塗り風、ダークファンタジー、近未来SF、低彩度 |

##### 悪い例・良い例

**悪い例：**

```text
かわいい魔法使いの女の子をいい感じに描いて
```

**問題点：**

- 「かわいい」「いい感じ」が曖昧
- 年齢・服装・構図・画風が指定されていない
- どこを直せば改善するのか分かりにくい

**良い例：**

```text
# 主題
10代の魔法使いの女の子。紺色のローブ、丸い帽子、短い杖を持っている。

# スタイル
アニメ調、太めの輪郭線

# 構図
正面からのバストアップ、明るい表情

# 仕上げ・条件
色数を抑える、背景はシンプル
```

**ポイント：**

- 4要素ごとに `#` でセクション分けされており、どこを直せばよいか一目で分かる
- 抽象語（「かわいい」「いい感じ」）が、観察可能な要素（服装・色・構図）に置き換わっている

##### 文章型と要素整理型

画像生成プロンプトは、必ずしも「タグの羅列」だけが正解ではない。最初は、意味が分かる文章として書き、それを要素に分解して整理するとよい。

**文章型：**

```text
銀髪ショートで赤い瞳の10代女性キャラクター。
黒いマントを羽織り、正面からのバストアップ構図。
アニメ調で、色数を抑えた落ち着いた雰囲気。
背景はシンプル。
```

**要素整理型（Markdown構造）：**

```text
# 主題
銀髪ショート、赤い瞳、10代女性、黒いマント

# スタイル
アニメ調、色数を抑える

# 構図
正面、バストアップ

# 仕上げ・条件
落ち着いた雰囲気、背景はシンプル
```

**ツールに入れるプロンプト例（最終フラット版）：**

```text
銀髪ショートで赤い瞳の10代女性キャラクター。黒いマントを羽織っている。正面からのバストアップ構図。アニメ調。色数を抑えた落ち着いた雰囲気。背景はシンプル。
```

#### 5. ワーク①：シンプルな1枚を出してみる（約30分）

- 教員から配布する **超シンプルなお題** を1枚出力する
  - 例：「画用紙の上にりんごが1個」
  - 例：「夕方の街角に立っている黒猫」
  - 例：「白い背景に、赤いスニーカーが1足」

- 手順：
  1. 4要素（主題／スタイル／構図／仕上げ・条件）を意識して短いプロンプトを書く（5分）
  2. 画像生成AIに投げる（5分）
  3. 出てきた絵を見て、何が思い通りで何がズレたかを **言葉で書き出す**（10分）
  4. 1か所だけ直して再投入（5分）
  5. グループで「変えた1か所」と「変化」を共有（5分）

##### 「1か所だけ変える」とは？

- 主題だけ変える：黒猫 → 白猫
- 構図だけ変える：バストアップ → 全身
- スタイルだけ変える：アニメ調 → 水彩風
- 条件だけ変える：背景あり → 背景なし
- 雰囲気だけ変える：明るい → 不思議な雰囲気

一度に複数変えると、どの言葉が結果に影響したのか分からなくなる。画像生成も、**1か所ずつ変えて観察する** のがコントロールの基本。

#### 7. 画像生成での「反復」のしかた（約15分）

- 第5回の反復テクニック（追加／削除／置き換え／比較）は、画像でも同じく効く
- ただし画像ならではのコツ：
  - **言葉で変えにくい部分** がある
    - 例：顔の特定パーツの微調整、指の形、左右の細かい差など
    - これらは後の回で、画像編集・ペイントオーバー・参照画像などとあわせて扱う
  - **1回ごとに違う絵** が出るので、「変えたつもりがランダム差」のことがある
    - 対策：**同じプロンプトで2〜3枚** 出して「ベースの揺れ」を見ておく
- 「育てる」の単位：
  - 1か所変更 → 数枚出す → 傾向を読む → 次の1か所
- 画像生成は **思考実験ではなく実験室** ：仮説 → 試行 → 観察 → 修正

##### 観察メモの例

| 観察する項目 | メモ例 |
|---|---|
| 思い通りだった点 | 銀髪、赤い瞳、バストアップは反映された |
| ズレた点 | マントが黒ではなく紫になった。背景が派手すぎた |
| 次に変える1か所 | 「背景は白でシンプル」を追加する |
| 変えない点 | 主題・構図・スタイルはそのまま |

#### 8. 比率・枚数・解像度の基本（約10分）

- ツールごとに細かい操作は違うが、共通の考え方を押さえる：
  - **アスペクト比**：縦長（人物）／横長（背景）／正方形（SNS・アイコン用）
  - **枚数**：1回で複数枚出して比較するのが基本
  - **解像度**：最初は中程度で量をこなし、良いものだけ高解像度で出し直す
- 「いきなり最高画質で1枚」より「中画質で数枚 → 良いものを高画質で再生成」
- ※詳しい構図コントロールは第7回

##### 第5回の設定文から画像プロンプトへ変換する例

**元の設定文：**

```text
このキャラクターは、人見知りだが芯が強い魔法使いの少女です。
古い森に住んでいて、普段は静かですが、困っている人を見ると放っておけません。

```

**そのままだと画像に出にくい言葉：**

- 人見知り
- 芯が強い
- 困っている人を見ると放っておけない
- ミステリアス
- 優しい雰囲気

**見た目に変換する：**

| 抽象的な設定 | 見た目に変換した例 |
|---|---|
| 人見知り | 少し目線を外している、控えめな表情 |
| 芯が強い | 背筋を伸ばして立っている、しっかり杖を持っている |
| 古い森に住んでいる | 背景に深い森、苔のある木、古い本 |
| ミステリアス | 暗めの青緑の色調、柔らかい逆光 |
| 優しい雰囲気 | 穏やかな表情、丸みのあるデザイン |

**画像生成用プロンプト：**

```text
# 主題
10代の魔法使いの少女。深い森の中に立っている。
紺色のローブ、丸い帽子、古い本を抱えている。
背筋を伸ばして短い杖を持っている。

# スタイル
アニメ調、丸みのあるデザイン

# 構図
正面からのバストアップ、少し目線を外した控えめな表情

# 仕上げ・条件
暗めの青緑の色調、柔らかい逆光、
背景は森だが、人物が目立つようにシンプルにする
```

##### 注意

- **実在の作品名・作家名は使わない**（第3回ルール）
- **狙い通りに出ない＝失敗ではない**。何が効かなかったかが学び
- 今回は「一貫性のあるキャラ」「複数ポーズ」は **狙わない**（第9回で扱う）
- 文字を画像内に入れる指定は、崩れやすい場合があるため今回は扱わない
- 細かい手指・装飾・左右差の制御は難しい場合があるため、今回は大きな方向性を合わせることを優先する
