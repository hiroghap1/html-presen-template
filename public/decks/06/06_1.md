# 画像生成AIの正体

---

# テキストから画像が出る仕組み（ざっくり）

- 大量の **「画像 ＋ それを説明するテキスト」** のセットを学習している
  - 言葉と色・形・構図の結びつきを学んでいる
- プロンプトを入れると、関係の深い特徴をもとに **新しい画像** を生成する
- ノイズのような状態から **少しずつ画像を作る**（Stable Diffusion など）
- 学習データに少ない概念・曖昧な言葉は **反映されにくい**

---

# 「コピペ」ではなく「再構成」

- 既存の絵をそのまま貼り合わせているわけではない
- 学習データの **特徴（質感・配色・構図の傾向）** をもとに新しい絵が出てくる
- だからこそ：
  - **実在の作家名・作品名** を入れると、その特徴に強く寄ってしまう（第3回の倫理リスク）
  - **無名でも具体的な記述** を積むのが正攻法

---

# 同じプロンプトでも、出るたび違う絵が出る

- 画像生成には **ランダム性** がある
- だから：
  - 1回出して「うまくいかなかった」と決めつけない
  - 同じプロンプトで **複数枚** 出して傾向を見る
- 「ガチャ」ではなく **「傾向のコントロール」**

---

# 画像生成は1回が「重い」

- チャット型：1回投げるコストが低い → 軽く何度も投げられる
- 画像生成：1回ごとに **時間・計算量** がかかる
- だから：
  - **作り込んでから投げる**
  - 何が効いて何が効かないかの **感覚を早めに掴む**

---

<!-- _class: lead -->

# チャット型と画像生成、何が違う？

---

# 入出力と「効きやすさ」の違い

| 観点 | チャット型 | 画像生成 |
|---|---|---|
| 入力 | テキスト | テキスト（＋画像） |
| 出力 | テキスト | 画像 |
| 役割指定 | 効きやすい | 効きにくい場合がある |
| 出力形式 | 自由に指定 | 画像（比率は別指定） |
| 反復のしやすさ | 言葉で細かく直せる | 微調整はしづらい |
| 1回のコスト | 低い | 高い |

---

# だから書き方も変わる

- チャット型は「あなたは○○の専門家です」と **長い文章** で組む
- 画像生成では：
  - プロンプトを **作り込んでから投げる**
  - **何が効いて何が効かないか** の感覚を早めに掴む
  - 出力結果を見て、**言葉に戻して観察** する
- どちらも **第5回の「解像度」の考え方は同じ**

# 画像生成プロンプトの基本要素

---

# 4つの要素

第5回の5要素は画像生成でも使える。
ただし画像生成では特に次の4要素が効く：

1. **主題**（Subject）— 何を描くか
2. **スタイル**（Style）— どんな画風で
3. **構図・視点**（Composition）— どう写すか
4. **仕上げ・条件**（Quality/Details）— どんな仕上がりにするか

---

# ①主題（Subject）

**何を** 描くか

### 例
> ファンタジー世界の10代の少女
> 夕方の街角に立っている黒猫
> 木製の机の上に置かれたりんご

- 「何か」「誰か」が抜けていると、AIは迷う
- 第4回の **観察可能な要素**（外見の具体）をここに乗せる

---

# ②スタイル（Style）

**どんな画風** で描くか

### 例
> アニメ調 ／ 水彩風 ／ フラットイラスト ／ 絵本風 ／ レトロなポスター風

- 「○○先生風」「○○（既存作品）風」は **避ける**（第3回）
- 代わりに **画材・タッチ・時代感** で指定する

---

# ③構図・視点（Composition）

**どう写すか** を指定する

### 例
> バストアップ ／ 全身 ／ 横顔 ／ 正面 ／ 俯瞰 ／ ローアングル
> 中央配置 ／ 余白を広く取る ／ 背景を大きく見せる

- 同じキャラでも、構図が変わると別の絵になる
- 今日は基本だけ。**詳しい構図コントロールは第7回**

---

# ④仕上げ・条件（Quality / Details）

**どんな仕上がりにするか** ＋ **使いどころの条件**

### 例
> 線画ベース ／ 色数を抑える ／ 柔らかい光 ／ 背景はシンプル
> 小学生向け教材に使えるように、怖すぎない表情

- 「高品質」「最高傑作」「いい感じ」だけでは曖昧
- 自分の **狙い** に対応する観察可能な語で書く

---

# 第5回テンプレとの違い：`#` で書く

第5回の `【役割】` ではなく、第6回からは **Markdownの `#` 見出し** で構造化する

```text
# 主題
ファンタジー世界の10代の少女

# スタイル
アニメ調
```

---

# なぜMarkdownにするのか

- LLM（ChatGPT、Claude等）は **Markdownを大量に学習** していて、`#` をセクション境界として正確に認識する
- `【】` は日本語圏ローカルな慣習で、AIによっては構造の読み取りが安定しない
- 画像生成AIに直接渡しても、`#` は害になりにくい
- ツールによっては、最終的に `#` を外して **自然文 or カンマ区切り** に整えて入力する

---

# 悪い例

```text
かわいい魔法使いの女の子をいい感じに描いて
```

### 問題点
- 「かわいい」「いい感じ」が **曖昧**
- 年齢・服装・構図・画風が指定されていない
- どこを直せば改善するのか分かりにくい

---

# 良い例

```text
# 主題
10代の魔法使いの女の子。紺色のローブ、丸い帽子、短い杖を持っている。

# スタイル
アニメ調、太めの輪郭線

# 構図
正面からのバストアップ、明るい表情

# 仕上げ・条件
色数を抑える、背景はシンプル
```

→ 4要素ごとに `#` で分かれていて、**どこを直せばいいか一目で分かる**

---

# 文章型と要素整理型

最初は「意味の通る文章」で書き、必要に応じて要素に分解する

### 文章型
```text
銀髪ショートで赤い瞳の10代女性キャラクター。
黒いマントを羽織り、正面からのバストアップ構図。
アニメ調で、色数を抑えた落ち着いた雰囲気。背景はシンプル。
```

---

# 要素整理型（Markdown構造）

```text
# 主題
銀髪ショート、赤い瞳、10代女性、黒いマント

# スタイル
アニメ調、色数を抑える

# 構図
正面、バストアップ

# 仕上げ・条件
落ち着いた雰囲気、背景はシンプル
```

---

# 実在作家・作品名を避けるための言い換え

| 避けたい指定 | 代わりに使える表現 |
|---|---|
| ○○先生風 | 太い輪郭線、淡い色、柔らかい陰影、繊細な線画 |
| ○○の作品風 | セル画風、絵本風、レトロなポスター風、フラットイラスト |
| 有名アニメ映画風 | 手描き感のある背景、温かい色調、やわらかい光 |
| 有名ゲーム風 | ドット絵、厚塗り風、ダークファンタジー、低彩度 |

---

# 4要素は「全部入れる」が目的じゃない

- 短くてもいい
- 大事なのは：
  - 出力がズレたとき **どの要素が足りない／曖昧か** を見直せること
- 第5回と同じ考え方：**フレームは思考の道具**

# 使うツール

## ChatGPT と Adobe Firefly

---

# 今回使う2つのツール

### ChatGPT（画像生成機能）
- チャット欄に **日本語の自然文** で指示できる
- 会話の流れで微修正を依頼できる
- 1回に1枚出てくる

### Adobe Firefly
- ブラウザ上の専用UI（firefly.adobe.com）
- アスペクト比・スタイル・参照画像などを **画面で指定** できる
- 1回に4枚出てくる
- Adobe Stock等の許諾済みデータで学習＝商用利用しやすい

---

# ChatGPTで画像を生成する手順

1. ChatGPTを開く（画像生成に対応した標準モデル）
2. チャット欄に **画像を生成して**＋プロンプトを書いて送信
3. 画像が1枚出てくる（数十秒）
4. 直したいところを **会話で追加指示**
   - 「もう少し背景を暗くして」
   - 「マントを赤にして、構図はそのまま」
5. 画像を右クリック／長押しで保存

---

# ChatGPTに渡すプロンプトの例

そのまま `#` 構造のプロンプトを貼ってOK。冒頭に **「以下の内容で画像を生成してください」** を付けるとブレが減る。

```text
以下の内容で画像を1枚生成してください。

# 主題
画用紙の上に、赤いりんごが1個。

# スタイル
水彩風、淡い色

# 構図
真上から、中央に配置

# 仕上げ・条件
背景は白、影は柔らかく
```

---

# ChatGPTの特徴と注意

### 得意なこと
- 自然文の指示理解 ／ 会話で少しずつ寄せる修正
- 「言葉での反復」が一番やりやすい

### 弱いところ・注意
- 1回に1枚しか出ない＝「ベースの揺れ」を見にくい
- 細かい構図やアスペクト比の指定は届きにくいことがある
- 著名キャラ・作家名はブロックされる／弱められる

---

# Adobe Fireflyで画像を生成する手順

1. ブラウザで **firefly.adobe.com** を開く（学校アカウントでログイン）
2. 左メニューから **「テキストから画像生成」** を選ぶ
3. 下のプロンプト欄にテキストを入力
4. 右側のパネルで **アスペクト比** と **コンテンツタイプ** を選ぶ
   - 縦長／横長／正方形
   - 写真／アート／グラフィック
5. **生成** → 4枚出てくる
6. 気に入った1枚にカーソルを合わせて **ダウンロード**

---

# Fireflyに渡すプロンプトの例

Fireflyの入力欄は **1つのテキストボックス** なので、最終的に **自然文 or カンマ区切り** に整える

### 入力欄に入れるテキスト（最終フラット版）
```text
画用紙の上に置かれた赤いりんごが1個。真上からの構図、中央に配置。水彩風、淡い色合い。背景は白、柔らかい影。
```

`#` 見出しのまま貼ってもある程度動くが、Fireflyは **自然文の方が安定** する傾向がある

---

# Fireflyの特徴と注意

### 得意なこと
- 1回で **4枚並ぶ** → ベースの揺れを見やすい
- アスペクト比・スタイルの指定が **UIで明確** にできる
- 商用利用のしやすさ

### 弱いところ・注意
- 会話での修正はできない＝**プロンプトを書き直して再生成**
- 日本語の細かいニュアンスは英語より弱いことがある
- 著名キャラ・実在人物は強く制限される

---

# 使い分けの目安

| やりたいこと | 向いているツール |
|---|---|
| 自然文で会話しながら寄せたい | ChatGPT |
| 4枚並べて揺れを見たい | Firefly |
| アスペクト比をきっちり指定したい | Firefly |
| 「もう少し○○」と細かく直したい | ChatGPT |
| 商用や配布を想定 | Firefly |

今回のワークでは **両方** 触る

# ワーク①：シンプルな1枚を出してみる

---

# ワークの目的

- 画像生成のクセを **小さなお題** で体感する
- 4要素を意識して、最初から **作り込んでから投げる** 練習
- ChatGPT／Fireflyの両方の **操作感** を掴む
- 「1回で完成しない」ことを前提に動く

---

# お題（教員が配布）

次のうちどれか1つを選んで、1枚出力する

- 画用紙の上にりんごが1個
- 夕方の街角に立っている黒猫
- 白い背景に、赤いスニーカーが1足

※ キャラクターはまだ出さない（次のワークでやる）

---

# やること（前半）

### 1. 4要素を意識して短いプロンプトを書く（5分）
- 主題／スタイル／構図／仕上げ・条件

### 2. ChatGPTに投げる（5分）
- 1枚出して、保存

### 3. 同じプロンプトをFireflyに投げる（5分）
- 4枚出して、保存

---

# やること（後半）

### 4. 出てきた絵を「言葉で」観察する（10分）
- 何が思い通り／何がズレたか、ツール間の差も書き出す

### 5. 1か所だけ直して再生成（5分）
- ChatGPTなら会話で追加指示／Fireflyならプロンプト書き直し

### 6. グループで「変えた1か所」と「変化」を共有（5分）

---

# 「1か所だけ変える」とは？

- 主題だけ：黒猫 → 白猫
- 構図だけ：バストアップ → 全身
- スタイルだけ：アニメ調 → 水彩風
- 条件だけ：背景あり → 背景なし
- 雰囲気だけ：明るい → 不思議な雰囲気

一度に複数変えると、どの言葉が結果に影響したのか分からなくなる。
画像生成も、**1か所ずつ変えて観察する** のがコントロールの基本

---

# 学び：画像でも「1か所ずつ」

- 大きく書き換えると、何が効いたか追えない
- **1か所だけ変える → 数枚出す → 観察** が基本サイクル
- これは第5回の「反復で育てる」と完全に同じ姿勢
