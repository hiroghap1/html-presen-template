# 今回は「構図」を深掘りする

第6回の4要素のうち、今回は **構図** にフォーカス

### Fireflyの強み：構図は2つの方法で指定できる
1. **プロンプトの言葉**
2. **GUIの設定**（縦横比・カメラアングル・構図参照）

→ **両方を組み合わせる** のが基本

---

# まず言葉を持つ：構図のボキャブラリー

| 区分 | 言葉 | 意味 |
|---|---|---|
| ショットサイズ | クローズアップ／バストアップ／全身／引き（ワイド） | 被写体までの距離・写る範囲 |
| アングル | アイレベル／俯瞰（上から）／あおり（下から） | カメラの高さ・角度 |
| 視点・向き | 正面／斜め45度／横顔／後ろ姿 | キャラの向き |
| 配置 | 中央／三分割で寄せる／余白を活かす | 画面内の置き方 |

---

# FireflyのGUIで指定できる主な項目

| 設定 | できること | 構図への効き方 |
|---|---|---|
| **縦横比** | 正方形／横長／縦長／16:9 | 縦長＝立ち姿、横長＝背景込み |
| **カメラアングル** | クローズアップ／ワイド／俯瞰／あおり 等 | ショットサイズ・アングルをワンタッチで |
| **構図参照** | 参照画像の輪郭・奥行きに合わせる | 「この配置で」を画像で指定 |
| 効果（Effects） | 質感・テクニック・雰囲気のプリセット | **仕上げ用**：構図ワークでは触らない |

# 言葉とGUI、どう使い分ける？

### 大まかな画角
→ **縦横比・カメラアングル（GUI）** で素早く

### 細かな向き・配置
→ **プロンプトの言葉** で補う

### 「この構図で出したい」が明確
→ **構図参照** に手描きラフや写真を入れる

※ 参照画像は **自分で描いたラフ／自分で撮った写真／配布素材** を使う
※ ネット画像や他人の作品を **無断で** 参照に使わない（第3回ルール）

# 悪い例

```text
いい感じのかっこいい構図で
```

### 問題点
- 「いい感じ」「かっこいい」が **曖昧**
- 距離・角度・配置が指定されていない
- どこを直せばいいか分からない

---

# 良い例（プロンプト＋GUI）

### プロンプト
```text
ローアングルから見上げる全身、斜め45度、
画面やや左に寄せて右に余白。
```

### Fireflyの設定（GUI）
- 縦横比：**縦長**
- カメラアングル：**あおり／全身が入る画角**

→ 言葉とGUIで **同じ方向** を指している

---

# 引き算（1限目）と組み合わせる

### 「全身を見せたい」とき
- 縦横比＝**縦長**
- カメラアングル＝**全身**
- プロンプトに「**見切れない**」

### 「人物に集中させたい」とき
- プロンプトに「**余白を広く・無地の背景**」
- 除外：「**他の人物は描かない**」

---

# 今日は触らない領域：後修正系

### Fireflyの「後修正」機能
- **生成消去**（要素を消す）
- **生成塗りつぶし**（部分的に作り直す）
- **生成拡張**（キャンバスを広げる）

### なぜ今日（あえて）触らないか
- 今回のゴールは **プロンプトで引き算・狙い撃ち** を体験すること
- 後修正は便利だが、**プロンプト力が育たない** 逃げ道になりやすい

### でも最後に少しだけのぞき見
→ ワーク②の清書後、**生成消去の実演（3分・任意）**
→ 本格運用は **第11・12回**「破綻の発見と修正」

### 4. 除外も併用（10分）

# やること（後半：清書 ― 5分）

### 6. 清書モデルに切り替えて、その1枚だけ清書
- 用途で選ぶ：
  - **商用・公開前提** → Image 5（Adobeモデル）
  - **複雑な指示・画像内テキストを忠実に** → GPT Image 2

### 切り替え時の注意
- Image 4で決めた **構図の言葉** は、**プロンプト本文に必ず書き写してから** 切り替える
  - 清書モデルは「カメラアングル」項目を持たない
- 縦横比・参照画像は清書モデル側でもそのまま使える
- 生成は **1〜2回まで** に絞る（クレジット消費が大きい）

### ※ 清書とは
- 同じ方針で **再生成** すること。**完全に同じ画像の高解像度化ではない**
- 顔・服・背景・画風（アニメ調→写実寄り）が変わることがある
- 違和感があればもう一方のモデルを試す／Image 4のまま提出も可（第8回で選定）
