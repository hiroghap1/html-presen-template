# 第4回：キャラクター設定の言語化

## 授業の概要

第1〜3回でAIの仕組み・使い方・倫理を押さえた上で、ここからは「AIに何を伝えるか」という制作の入口を扱う。AIにとって入力は基本的に**言葉**であり、頭の中の曖昧なイメージを言葉にできなければAIにも伝わらない。本回は画像生成やプロンプト構築に入る前段として、シナリオや一行設定から **性格・背景・外見的特徴** を解像度高く言語化するトレーニングを行う。「可愛い」「カッコいい」では伝わらない。**誰が見ても同じ絵が思い浮かぶ言葉**を選べるようになることがゴール。

## 授業目標

1. キャラクター設定を「外見／内面／背景／関係性」の観点で構造的に言語化できる
2. 抽象語（可愛い・強そう）を、具体的な観察可能な要素に置き換えられる
3. 一行の発想から、AIに渡せる粒度の設定シートまで膨らませることができる

## 授業内容

### 【1時間目】90分

#### 1. 出席確認・前回の振り返り・前回の理解度復習クイズ（約10分）

- 出席確認しながら、理解度復習クイズ
- 前回キーワード復習：「依拠性／類似性」「3つの段階（学習・生成・利用）」「自分のAIガイドライン」
- 今日のキーワード予告：「言語化」「解像度」「観察可能な特徴」「設定シート」

#### 2. なぜ「言葉」が重要なのか（約15分）

- AIへの入力は基本的に言葉（テキスト）
- 「頭の中にあるイメージ」と「言葉になっているイメージ」のギャップ
- 自分の中で曖昧なものは、AIにも曖昧にしか伝わらない
- 言語化は **AIのため** だけでなく **自分の理解の整理** でもある
- 第2回の「整理補助」とつながる：自分の中にあるものを取り出して並べる作業

- 例：同じキャラを言葉で説明（実際に時間をとって描かせてみてもいいかも。バストショットラフ画）
  - 「黒髪のクールな男子高校生」 → 何通りもの解釈ができる
  - 「短く切り揃えた黒髪、切れ長の目、無表情、制服のネクタイを緩めて着崩している男子高校生」 → 像が定まる

#### 3. キャラクター設定の4つの層（約25分）

設定を **層で分ける** ことで漏れと重複を防ぐ。

1. **外見**：髪・目・体型・服装・身につけているもの・年齢
2. **内面**：性格・口調・好きなもの嫌いなもの・癖・価値観
3. **背景**：出身・家族構成・経歴・現在の状況・所属
4. **関係性**：物語の中での役割・他キャラとの距離・対立構造

- 各層は独立ではなく、互いに矛盾しないか・補強し合うかが重要
  - 例：「箱入り娘の貴族令嬢」（背景）と「鋲付きライダースジャケット」（外見）は **意図的な対比** か **設定ミス** か？
- 商業のキャラクター設定資料（モデルシート）を例に、何が言語化されているかを観察
- 抽象語禁止トレーニング：
  - NG「可愛い」 → OK「丸い大きな目、頬の赤み、小柄」
  - NG「強そう」 → OK「眉間に縦皺、肩幅が広い、握りしめた拳」
  - 「観察できる特徴」に置き換える

#### 4. ワーク①：抽象語を具体に分解する（約40分）

- お題（キャラクターイラストをこちらで用意）
- 各イラストに **抽象的な一言タグ** が付いている（例：「優しそう」「怖そう」「賢そう」「天然っぽい」「ミステリアス」）
- ワーク手順：
  1. なぜそう見えるのか、**観察可能な要素** をスプレッドシートに5つ以上書き出す（10分）
  2. 同じ抽象語に対して、別の見た目でも成立する例を1つ考える（10分）
  3. グループで共有 → なぜ印象が一致したか／ズレたかを話す（15分）
  4. 数組に発表してもらう（5分）
- 学び：「印象」は **要素の集合** から生まれる／要素を入れ替えれば印象もコントロールできる

- 今後のレッスン
  - 食事の際、「美味しい」以外の言葉で食べたときの感情を表現する
  - 街中の広告などで「かわいい」「かっこいい」「すごい」「いまいち」と感じた時、なぜそう感じたのか言葉で表現する

### 【2時間目】90分

#### 5. 出席確認（約5分）

- 出席確認

#### 6. 一行から設定を膨らませる手順（約20分）

ゼロから無限に書こうとすると手が止まる。**質問に答える形** で順に埋めていく。

- スタート：一行の核（例：「人間嫌いの図書館司書」）←こちらでやってみせる
- ステップ：
  1. **役割**：物語上どんな立ち位置か（主人公／相棒／敵対／日常の脇役）
  2. **時代・世界**：いつ、どこにいる？（現代日本／架空のファンタジー／近未来都市）
  3. **外見の決め手**：見た瞬間に印象を作る要素3つ（髪・服・小物・体型・表情のうちどれを強調？）
  4. **内面の核**：たった1つだけ大切にしているもの／許せないもの
  5. **過去の出来事**：今のキャラを形作った過去の事件を1つ
  6. **矛盾／ギャップ**：意外性を1つ仕込む（例：強面なのに猫好き）
- 第2回でやった「AIに壁打ち」の応用：自分で書いた素案をAIに「不足している観点を指摘して」と投げると、抜けている層が見える

#### 7. AIに渡す前提のフォーマット（約15分）

- AIは **箇条書き＋カテゴリ分け** に強い／長文小説体は読み落とされやすい
- 推奨フォーマット例（次回プロンプトで再利用する）：

  ```
  【名前】
  【年齢／性別／種族】
  【外見】髪型・目・体型・服装・小物（5〜10項目）
  【性格】3〜5語＋一行説明
  【背景】出身・経歴・現在の状況（3行以内）
  【口調・喋り方】一人称・語尾・特徴的な言い回し
  【物語上の役割】
  ```

- 第3回の倫理を踏まえた注意：
  - 既存キャラの名前・固有名詞をそのまま入れない（依拠性が認められやすくなる）
  - 「○○先生風」「○○みたいな」と特定作家・作品を指す表現は避ける
  - **属性の組み合わせで個性を作る** という思考に切り替える

#### 8. ワーク②：一行からの設定シート作成（約40分）

- 各自、以下のいずれかの一行から設定シートを作る（自分で考えてもよい）
  - 「砂漠の街で道具を売る少女」
  - 「人間嫌いの図書館司書」
  - 「廃墟の遊園地に住む元アイドル」
  - 「夜だけ営業する古書店の店主」
  - 「機械化された街で唯一の生身の医者」
- 手順：
  1. 一人で素案を15分で書く（上のフォーマットを埋める）
  2. ChatGPTに「この設定で抜けている観点を5つ指摘して」と投げる（10分）
  3. 指摘を踏まえて加筆・修正（10分）
  4. スプレッドシートに完成版を貼り付け、近くの人と読み合いをする（5分）
- 評価軸（自己チェック）：
  - 抽象語に逃げず、観察可能な要素になっているか
  - 4つの層（外見・内面・背景・関係性）に大きな抜けがないか
  - 矛盾や違和感のあるギャップが意図的に入っているか

#### 9. まとめと次回予告（約5分）

- 本日のふりかえり：4つの層（外見/内面/背景/関連性）、抽象語の分解、一行から設定シートへ
- 次回予告：第5回「基本プロンプトエンジニアリング」（今日作った設定シートを、AIに正確に伝える書き方に変換していく）
- **今日の設定シートは次回も使うので保存しておくこと**

#### 10. 理解度クイズ（約5分）

- Q1. キャラクター設定をAIに伝える際、もっとも避けたい書き方は？
  - A. 抽象的な一言（「可愛い」「カッコいい」）だけで済ませる → **正解。** 解釈の幅が大きすぎてAIに像が伝わらない
  - B. カテゴリ別に箇条書きで具体的な要素を並べる → AIが読み取りやすい望ましい形
  - C. 観察可能な特徴（髪型・体型・表情）に分解して書く → 望ましい形
- Q2. キャラクター設定を構造的に整理するときの「4つの層」として、本講義で扱ったのは？
  - A. 外見・内面・背景・関係性 → **正解。**
  - B. 顔・身体・服装・武器
  - C. 主人公・敵・味方・モブ → 物語の役割の分類で、設定の層分けではない
- Q3. 既存作品の固有名詞や作家名を設定文に入れるのが望ましくないのはなぜ？
  - A. AIが処理できないから → 処理自体は可能
  - B. 第3回で学んだ依拠性が認められやすくなり、権利侵害につながるリスクがあるから → **正解。**
  - C. 文字数が増えてプロンプトが長くなるから → 本質的な理由ではない

#### 11. 振り返りシート記入（約10分）

- 振り返りシートの記入

## 授業の目標（詳細）

1. **キャラクター設定を構造的に言語化できる**（外見・内面・背景・関係性の4層で漏れなく組み立てる）
2. **抽象語を観察可能な要素に分解できる**（「可愛い」を構成要素に翻訳する力＝AIに伝わる言葉に翻訳する力）
3. **一行から設定シートまで膨らませられる**（質問形式の手順とAIへの壁打ちを使って、次回プロンプトに渡せる粒度まで具体化する）

## 次回への接続

第5回「基本プロンプトエンジニアリング」では、本回で作った設定シートを **AIへの指示文（プロンプト）** に変換していく。今日の段階では「人間が読んで分かる設定」、次回は「AIが正確に再現できる指示」へ。第6回からの画像生成の素材として今日の設定シートを再利用する。
